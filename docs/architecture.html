Architecture & Data Flow

One agent brain, two runtimes. A transaction generator feeds a live demo world; the same Penny agent that runs the graded bench powers the product. Executed actions land as notifications on the recipient's screen.

Bench runtime (parallel, graded): the same Penny agent, but data plane = McContext's hidden MCP, and there's no generator, console, or recipient screens — a simulator drives cases, the agent calls the typed submit_* action tools, and a judge scores the trace + outcome. Product work = bench work.
Dev-time harness (not in either runtime): the Eval harness uses generator ground-truth to score precision / recall / F1 per duty offline — the gate is don't spend a bench life until offline F1 ≥ target. Regression fixtures = the real finds (str_009 / str_003 cash, stf_009_6 loss) and the decoys (55 recurring, 4 sub-tolerance 3-way lines). A Cost governor (candidate-net-first, turn/token cap, model tiering) keeps scans inside the $50 budget.

2 · Data flow — product runtime

The precision funnel end to end: generate → sense (cheap, high recall) → adjudicate (judgment, high precision) → resolve → notify.

Transaction Generator emits a batch of txns/events — plus any injected leaks/decoys — into the World Store. demo control
A scan fires — Scan now, schedule, or new-batch trigger — waking the Orchestrator.
Orchestrator invokes the Penny agent; it reads the store through the MCP read tools.
Candidate nets (deterministic tools) surface everything suspicious — high recall (e.g. 365 anomalies). sensing
Adjudication (LLM judgment) reasons only on candidates: real vs decoy, root cause, or abstain — high precision (365 → ~3). deciding
For each verdict the agent calls an action tool under graduated autonomy (open_lp_case, hold_payment, draft_credit_memo, record_flag).
Action Executor gates it (auto / 1-click / approval-required), produces the artifact, writes the audit log. Money & accusations never fire autonomously.
Notification Service pushes the result to the recipient screen 🔔 (Regional Manager / Supplier / AP) + a console toast.
Controller triages in the Console (confirm / dismiss / approve); feedback + outcomes loop back to the audit trail.

3 · Data flow — bench runtime

Same brain, stripped to what's graded.

Bench harness starts a session with the registered agent id + version. 3 lives
A simulator presents hidden cases; the agent investigates via the McContext MCP read tools.
Same funnel: candidate nets → adjudication → verdict / abstain.
Agent records the outcome through action tools — this is the graded result.
A judge scores the trace + outcome on precision, recall, judgment, security, efficiency, communication.
Why the generator matters for winning: the bench uses hidden data we can't see, and the world DB is static — so for the product demo the generator lets us stage the exact narrative on stage: inject a skim → hit Scan → watch Penny catch it, clear the decoys, open the LP case, and the Regional Manager's screen light up. Live, controllable, repeatable.
